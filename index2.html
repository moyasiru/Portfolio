프로젝트 배경

SSO 및 ERP 연동 개발을 진행하면서 토큰 기반 인증 방식이 있다는 것을 알았고, 보다 체계적이고 안전한 인증 처리를 위해 Spring Security의 필요성을 느껴 학습하게 되었습니다.

개발 환경
Java Spring Boot Hibernate MySQL Spring Security
주요 구현 내용
1. 통합 소셜 로그인
Spring Security를 활용하여 카카오, 구글, 네이버 소셜 로그인 기능을 통합적으로 구현하였으며, 기존 로그인 방식(Local Login)과도 연동되도록 설정
2. 보안 강화
사용자 인증 과정에 보안을 한층 높였으며, 민감정보를 BCrypt 기반으로 암호화
3. 확장 가능한 구조
다양한 소셜 로그인 기능을 통합 관리할 수 있도록 하나의 메소드 안에서 각 플랫폼별 인증 과정을 처리하는 공통 모듈을 설계 및 개발. 코드의 중복을 줄이고 유지보수성을 높였으며, 신규 소셜 플랫폼 추가 시에도 손쉽게 확장할 수 있는 구조를 구현
Spring Security 개념 및 구현 방법

개발 환경
Java Spring Boot MyBatis Oracle Thymeleaf Apache Kafka
주요 구현 기능
1. Amchart를 이용한 데이터 시각화
Amchart 라이브러리를 활용하여 동적 차트를 구현하고, DB와 연동하여 실시간 데이터를 기반으로 차트를 생성
2. Slick 슬라이더 플러그인 활용
Slick Slider 플러그인을 사용하여 반응형 이미지 슬라이더를 구현, 웹 페이지의 디자인을 상호작용성 있게 만들어 사용자 경험을 향상
3. Jsoup을 이용한 웹 페이지 크롤링
Jsoup 라이브러리를 사용하여 외부 웹 페이지를 크롤링하고, 필요한 데이터를 추출하여 동적 콘텐츠를 표시하는 기능을 구현
4. Kafka와 웹소켓을 연동한 채팅
웹소켓만으로는 세션 단위로 메시지를 전달하기 때문에 하나의 채팅방에만 메시지를 보내는 데 한계가 있었습니다. 이를 해결하기 위해 Kafka를 사용하여 여러 채팅방에서 메시지를 효율적으로 처리하고 분배할 수 있도록 개선
5. CKEditor 에디터 활용
CKEditor를 이용하여 텍스트 크기, 문단 나눔, 사진 업로드 등의 기능을 편리하게 이용 가능하게 구현
